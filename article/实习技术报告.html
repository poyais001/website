问题背景

问题导入：叶片表面的网格生成

  对风机叶片生成表面网格时，需要将叶片曲面的一部分映射到平面内进行剖分，再将剖分好的网格映射回曲面。由于叶片往往是狭长的，为了使最终的网格变形较小，在映射到平面时往往选择映射到狭长的四边形或三角形内。

  在实际应用中，需要向叶片上铺设碳纤维布、玻璃纤维布等材料。为了适应各向异性材料的性质，更好地计算材料在受力时的状态，在网格生成时需要确保大部分网格由较为规整的四边形构成。

整体问题

  由于边界两侧宽度不等，用结构化网格会导致靠近边界处网格质量的下降，用非结构网格又不能较好地表现各向异性材料的性质。为了使网格以四边形为主，同时保持整体尺寸一致，我们采用了逐层生成四边形网格，在收窄区域加入三角形过渡的方法。

  在本次实习中，本组以GridFrame网格框架为基础，通过分层生成，三角过渡的方式，实现了平面狭长区域的网格生成。在完成了基础的需求之后，我们又为其添加分区生成、分层分区生成等不同的功能，以达到更好的网格质量，满足不同的需求。

Part 1 - 实现过程说明

在 main() 中新建StripDivide对象，并使用HTPoint点进行初始化。

HTMesh::HTPoint p0(0, 0, 0), p1(10, 1, 0), p2(10, 4, 0), p3(0, 5, 0);  
HTMesh::StripDivide new_mesh(p0, p1, p2, p3);

根据点的个数，构造函数会调用 initial_tri() 或 initial_rec() 分别初始化三角形和四边形边界。点的顺序不能打乱，这决定了之后的剖分方向。

使用 set_rec() 设置网格尺寸，参数是单个矩形网格的长和宽。

new_mesh.set_rec(0.25, 0.15);

使用 set_part_scale() 设置分区比例，这在分区划分网格中会用到。
参数1是从三角形边界分区时顶部三角形的分区比例，参数2是从短边划分出单侧的结构网格的比例。

new_mesh.set_part_scale(0.3, 0.4);

根据不同需求，可以调用不同的函数进行剖分，以实现不同的效果。

  • 使用 all_divide() 实现最基础的剖分。可用于三角形和四边形区域。
  • 使用 quad_parts() 对四边形区域进行分区剖分，改善边缘处的波浪纹。
  • 使用 tri_parts() 对三角形区域进行分区剖分。
  • 使用 quad_multi_parts() 对四边形区域进行分层分区剖分，在分区剖分的基础上收窄了不规则区域的范围。
  • 使用 refinement() 对四边形区域网格的中间区域进行单向加密。
//基础切分
new_mesh.all_divide();

//四边形两侧分区
new_mesh.quad_parts();

//三角形分区剖分
new_mesh.tri_parts();

//四边形分层分区
new_mesh.quad_multi_parts();

//中间单向加密（四边形）
new_mesh.refinement();

all_divide()

对四边形边界或三角形边界进行基础剖分。可以调用三种点连接方式：单侧推进/双侧推进/投影推进。
对初始边界，先进行一次底边切分，使其与后续输入一致。对于侧边上的每个点，进行一次 切条 -> 底边切分 -> 连接点 ，完成一层网格，最后（如果是四边形边界）再对顶层进行切分。

//! 进行全部切分过程
void StripDivide::all_divide()
{
  //计算侧边切分
  ClacSideStep();

  //对初始网格先进行一次底边切分
  divide_bottom();

  //对于侧边上每个点,进行一次
  //  切条 --> 底边切分 --> 连接两列点
  for (int i = 1; i < divide_num_side; i++)
  {
    slice();
    divide_bottom();
    //单侧推进/投影推进
    //connect(Grid.ccw_rotated_halfedge(Grid.opposite_halfedge(todo_side)));
    //双侧推进
    connect(Grid.ccw_rotated_halfedge(Grid.opposite_halfedge(todo_side)), Grid.opposite_halfedge(Grid.next_halfedge(down_side)));
  }

  //处理顶部的最后一条(仅当边界为梯形)
  if (Grid.ccw_rotated_halfedge(todo_side) != down_side)
  {
    divide_top();
  }
}

quad_parts()

对四边形边界进行分区剖分。分区分为三个区域：沿两个侧边的结构网格区域和中间的剖分区域。根据预设的分区比例在顶边截取一定长度的边并投影至底边，形成结构网格区域，因为上下边等长，对该区域调用 all_divide 即可生成结构网格。对于中间的区域，调用一次 all_divide 。
分区是将一个四边形拆分成了三个独立的四边形分别进行剖分。由于输出的STL文件只包含面而不包含拓扑关系，拆分面并不影响网格的整体性。
在将STL输入PointWise时，会对网格结构进行重建。如果两个点的距离大于容差，会被认为是两个点，进而导致网格内部产生边缘。为了避免这种情况，将 pmp::scalar 指定为 double 来提升精度，避免产生内部边缘。

// 以当前的todo_side为边界，根据给定的分区比例进行分区剖分
void StripDivide::quad_parts()
{
    //获取原四边形的四个顶点
    HTPoint p0 = Grid.position(Grid.to_vertex(down_side));
    HTPoint p1 = Grid.position(Grid.from_vertex(down_side));
    HTPoint p2 = Grid.position(Grid.from_vertex(todo_side));
    HTPoint p3 = Grid.position(Grid.to_vertex(todo_side));

    //计算顶边切分数量
    divide_num_top = calc_num(Grid.prev_halfedge(down_side), rec_wid);

    //计算顶边切分步长
    step_vector_top = calc_step(Grid.prev_halfedge(down_side), divide_num_top);

    //计算边缘矩形区的宽度（矩形数）
    int edge_part_wid = round(divide_num_top * scale_quad);

    //计算底边切分数量和步长
    ClacBottomStep();

    //插入的四个新点，用于分区成三个四边形
    HTPoint p4 = p0 - edge_part_wid * step_vector_bottom,
            p5 = p1 - edge_part_wid * step_vector_top,
            p6 = p2 + edge_part_wid * step_vector_top,
            p7 = p3 + edge_part_wid * step_vector_bottom;

    Grid.delete_face(Grid.face(todo_side));

    //对三个划分的四边形分别进行all_divide的过程
    //因为共用一个侧边切分数，仅在第一个quad计算，且不能使用all_divide
    initial_quad(p4, p5, p6, p7);
    {
        ClacSideStep();
        divide_bottom();
        for (int i = 1; i < divide_num_side; i++)
        {
        slice();
        divide_bottom();
        connect(Grid.ccw_rotated_halfedge(Grid.opposite_halfedge(todo_side)), Grid.opposite_halfedge(Grid.next_halfedge(down_side)));
        }
        divide_top();
    }

    initial_quad(p0, p1, p5, p4);
    {
        step_vector_side_up = calc_step(Grid.opposite_halfedge(todo_side), divide_num_side);
        step_vector_side_down = calc_step(Grid.opposite_halfedge(down_side), divide_num_side);
        divide_bottom();
        for (int i = 1; i < divide_num_side; i++)
        {
        slice();
        divide_bottom();
        connect(Grid.ccw_rotated_halfedge(Grid.opposite_halfedge(todo_side)), Grid.opposite_halfedge(Grid.next_halfedge(down_side)));
        }
        divide_top();
    }

    initial_quad(p7, p6, p2, p3);
    {
        step_vector_side_up = calc_step(Grid.opposite_halfedge(todo_side), divide_num_side);
        step_vector_side_down = calc_step(Grid.opposite_halfedge(down_side), divide_num_side);
        divide_bottom();
        for (int i = 1; i < divide_num_side; i++)
        {
            slice();
            divide_bottom();
            connect(Grid.ccw_rotated_halfedge(Grid.opposite_halfedge(todo_side)), Grid.opposite_halfedge(Grid.next_halfedge(down_side)));
        }
        divide_top();
    }
}

tri_parts

对三角形边界进行分区剖分，先切掉顶部的三角形，应用基础切分，再对下面的四边形应用 quad_part() 。

// 根据给定比例，对三角形进行分区，分别进行剖分
void StripDivide::tri_parts()
{
  HTPoint p0 = Grid.position(Grid.to_vertex(down_side));
  HTPoint p1 = Grid.position(Grid.from_vertex(todo_side));
  HTPoint p2 = Grid.position(Grid.to_vertex(todo_side));
  Grid.delete_face(Grid.face(todo_side));

  //插入的切分点
  HTPoint p3 = (1 - scale_tri) * p1 + scale_tri * p0;
  HTPoint p4 = (1 - scale_tri) * p1 + scale_tri * p2;

  //切分上侧三角形区域
  initial_tri(p3, p1, p4);
  all_divide();

  //切分下侧四边形区域
  initial_quad(p0, p3, p4, p2);
  quad_parts();
}

quad_multi_parts()

对四边形边界进行分层分区剖分。使用 quad_parts 时，在顶边确定分区宽度并投影至底边，会导致靠近底边方向波浪纹区域的范围扩大。为了在大部分区域保证网格的正交性，可以采用先纵向分层，再横向分区的方法，在每层分别进行 quad_parts ，使每一层的外侧都是结构网格，提升了整体的网格效果。

// 对四边形进行分多层的两侧切分
void StripDivide::quad_multi_parts()
{
    ClacSideStep();
    HTPoint vector_up = step_vector_side_up;
    HTPoint vector_down = step_vector_side_down;
    int num_side = divide_num_side;

    //计算每层的切分数量，向下取整
    int step_per_floor = divide_num_side / floors;

    //计算第一层的四个点
    HTPoint p0 = Grid.position(Grid.to_vertex(down_side));
    HTPoint p3 = Grid.position(Grid.to_vertex(todo_side));

    HTPoint p1 = p0 + vector_down * (num_side - step_per_floor * (floors - 1));
    HTPoint p2 = p3 + vector_up * (num_side - step_per_floor * (floors - 1));

    //删掉切分前的大面
    Grid.delete_face(Grid.face(todo_side));

    //剖分第一层
    initial_quad(p0, p1, p2, p3);
    quad_parts();

    //逐层前进，分别进行分区剖分
    for (int i = 1; i < floors; i++)
    {
        p0 = p1;
        p3 = p2;
        p1 += vector_down * step_per_floor;
        p2 += vector_up * step_per_floor;

        initial_quad(p0, p1, p2, p3);
        quad_parts();
    }
}

refinement()

对四边形边界进行剖分时，对中间部分进行单向加密。加密区的范围可以通过参数指定。自底向上进行剖分时，先生成底侧稀疏区，之后调整标准矩形宽边的尺寸生成加密过渡区、加密区、稀疏过渡区，之后生成上侧稀疏区。

//! 分区单向加密
void StripDivide::refinement()
{
    ClacSideStep();
    divide_bottom();

    int part1 = divide_num_side / 3, part2 = divide_num_side - part1 * 2;
    double origin_rec_wid = rec_wid;//加密区需要更改矩形尺寸，存储原尺寸

    assert(part2 > 5);

    //切分part1，未加密区域
    for (int i = 0; i < part1; i++)
    {
        slice();
        divide_bottom();
        connect(Grid.ccw_rotated_halfedge(Grid.opposite_halfedge(todo_side)));
    }

    //加密过渡层，第一层
    {
        rec_wid = origin_rec_wid / 1.4;
        slice();
        divide_bottom();
        connect(Grid.ccw_rotated_halfedge(Grid.opposite_halfedge(todo_side)));
    }

    //加密过渡层，第二层
    {
        rec_wid = origin_rec_wid / 2;
        slice();
        divide_bottom();
        connect(Grid.ccw_rotated_halfedge(Grid.opposite_halfedge(todo_side)));
    }

    //切分part2，加密区域
    for (int i = 0; i < part2 - 4; i++)
    {
        slice();
        divide_bottom();
        connect(Grid.ccw_rotated_halfedge(Grid.opposite_halfedge(todo_side)));
    }

    //稀疏过渡层，第一层
    {
        rec_wid = origin_rec_wid / 1.4;
        slice();
        divide_bottom();
        connect(Grid.ccw_rotated_halfedge(Grid.opposite_halfedge(todo_side)));
    }

    //稀疏过渡层，第二层
    {
        rec_wid = origin_rec_wid;
        slice();
        divide_bottom();
        connect(Grid.ccw_rotated_halfedge(Grid.opposite_halfedge(todo_side)));
    }

    //切分part1，稀疏区域
    for (int i = 0; i < part1 - 1; i++)
    {
        slice();
        divide_bottom();
        connect(Grid.ccw_rotated_halfedge(Grid.opposite_halfedge(todo_side)));
    }

    //处理顶部的最后一条(仅当边界为梯形)
    if (Grid.ccw_rotated_halfedge(todo_side) != down_side)
    {
        divide_top();
    }
}

initial_quad()

用于初始化梯形边界，以输入的四个点添加一个面，同时记录两个半边作为之后所有操作的索引。
initial_tri() 同理，用于三角形输入。

//初始化梯形边界
void StripDivide::initial_quad(HTPoint p0, HTPoint p1, HTPoint p2, HTPoint p3)
{
  HTVertex v0 = Grid.add_vertex(p0);
  HTVertex v1 = Grid.add_vertex(p1);
  HTVertex v2 = Grid.add_vertex(p2);
  HTVertex v3 = Grid.add_vertex(p3);
  Grid.add_quad(v0, v1, v2, v3);

  //存储侧边
  todo_side = Grid.find_halfedge(v2, v3);
  down_side = Grid.find_halfedge(v1, v0);
}

calc_num()

用于计算边的切分数量，用给定的边的长度除以目标边长，四舍五入得到切分数量。

//! 计算切分数量
//! \param he:   待切分的半边
//! \param len:  给定边长
//! \return 切分数量
int StripDivide::calc_num(HTHalfedge he, pmp::Scalar len)
{
  //半边的起止点
  HTPoint start = Grid.position(Grid.from_vertex(he));
  HTPoint end = Grid.position(Grid.to_vertex(he));

  //计算切分数量
  pmp::Scalar x = end[0] - start[0], y = end[1] - start[1];
  return round(sqrt(x * x + y * y) / len);
}

calc_step()

用于计算边的切分步长，用给定的半边除以切分数量得到切分步长，作为向量存储在Point中。

//! 计算切分步长
//! \param he: 待切分的半边
//! \param n: 切分数量
//! \return 切分步长
HTPoint StripDivide::calc_step(HTHalfedge he, int n)
{
  //半边的起止点
  HTPoint start = Grid.position(Grid.from_vertex(he));
  HTPoint end = Grid.position(Grid.to_vertex(he));

  //计算切分步长
  return (end - start) / n;
}

divide_edge()

用于对边进行切分，每次前进一个步长插一个点，点数可以指定。
注意是从半边终点出发向起点方向插点，好处是不需要更新坐标。步长向量step与输入的半边he方向相反。

//! 切分边,在给定半边的终点开始,向起点方向插n个点,he和step反向
//! \param he: 待切分的半边
//! \param n: 切分次数
//! \param step: 切分步长
void StripDivide::divide_edge(HTHalfedge he, int n, HTPoint step)
{
  //插入点的坐标，前进一个步长作为初始值
  HTPoint index = Grid.position(Grid.to_vertex(he)) + step;

  for (int i = 0; i < n; i++, index += step)
  {
    Grid.insert_vertex(he, Grid.add_vertex(index));
  }
}

ClacSideStep()

对于当前 todo_side 和 down_side 所指向的两个侧边，计算切分数量和切分步长，将计算结果存储到类中的私有变量中：

  • divide_num_side：切分数量
  • step_vector_side_up：上侧边切分步长
  • step_vector_side_down： 下侧边切分步长

注意上下侧边使用同一个切分数量，且这一步只计算，不进行切分。

 //! 计算侧边的切分数量和切分步长
void StripDivide::ClacSideStep()
{
  //计算侧边切分数量
  divide_num_side = calc_num(todo_side, rec_len);

  //计算上侧边切分步长
  step_vector_side_up = calc_step(Grid.opposite_halfedge(todo_side), divide_num_side);

  //计算下侧边切分步长
  step_vector_side_down = calc_step(Grid.opposite_halfedge(down_side), divide_num_side);
}

ClacBottomStep()

也是对基础函数的简单封装，对当前的底边，也就是 todo_side 的下一条半边计算切分，并将计算结果存储到类的私有变量中。

  • divide_num_bottom：底边切分数量
  • step_vector_bottom：底边切分步长
 //! 计算底边的切分数量和切分步长
void StripDivide::ClacBottomStep() 
{
  //计算底边切分数量
  divide_num_bottom = calc_num(Grid.next_halfedge(todo_side), rec_wid);

  //计算底边切分步长
  step_vector_bottom = calc_step(Grid.next_halfedge(todo_side), divide_num_bottom);
}

slice()

从梯形的底部切下一层，用于剖分第一层的网格。两侧都前进一个步长之后连接点。
insert_edge() 是在输入的两个半边的终点之间建立一条新边。插入的新边就是新的底边。

//! 从底部切片
//! 提供参数：侧边side，两个切分步长向量
//! 从上下侧边的端点分别出发寻找新端点，连接两个新端点
void StripDivide::slice()
{
  //进行一次切分
  divide_edge(todo_side, 1, step_vector_side_up);
  divide_edge(down_side, 1, step_vector_side_down);

  //插入新底边
  Grid.insert_edge(todo_side, Grid.opposite_halfedge(Grid.next_halfedge(down_side)));
}

divide_bottom()

对当前 todo_side 对应的底边进行切分，这个函数中包含了计算切分数的过程。

//! 切分底边(当前todo_side对应的底边)
void StripDivide::divide_bottom() 
{
  //计算切分数量和切分步长
  ClacBottomStep();
  
  //进行切分
  divide_edge(Grid.opposite_halfedge(Grid.next_halfedge(todo_side)), divide_num_bottom - 1, step_vector_bottom);
}

divide_top

到最后一层时，对顶边进行切分并连接该层的点。因为每前进一步对底边做一次切分，到达顶部后，需要对顶边单独做一次切分，过程与 divide_bottom() 相同。当初始边界为三角形时，最后一层只剩一个三角形，不需要进行切分。

 //! 切分顶部(仅当边界为梯形)
void StripDivide::divide_top()
{
  //确认有顶边（不是三角形）
  assert(Grid.ccw_rotated_halfedge(todo_side) != down_side);

  //计算顶边切分数量
  divide_num_top = calc_num(Grid.prev_halfedge(down_side), rec_wid);

  //计算顶边切分步长
  step_vector_top = calc_step(Grid.prev_halfedge(down_side), divide_num_top);

  //切分顶边
  divide_edge(Grid.prev_halfedge(todo_side), divide_num_top - 1, step_vector_top);

  //连接点
  connect(todo_side, Grid.opposite_halfedge(down_side));
}

connect_rec()

输入一条半边，连接前后两条半边的另一侧，构建一个矩形，返回新建边的外侧半边。

//在(四边形以上的)多边形中使用,连接前后两条边的末端,构建一个矩形
//返回新边的外侧半边
HTHalfedge StripDivide::connect_rec(HTHalfedge he)
{
  //he必须在大于四边形中
  assert(Grid.valence(Grid.face(he)) > 4);
  return Grid.insert_edge(Grid.prev_halfedge(Grid.prev_halfedge(he)), Grid.next_halfedge(he));
}

connect_tri()

输入一条半边，连接当前半边起点和下一条半边的终点，构建三角形，返回新建边的外侧半边。

//连接当前半边起点和下一条半边的终点，构建三角形
//返回新边的外侧半边
HTHalfedge StripDivide::connect_tri(HTHalfedge he)
{
  //四边形仍然可以插入三角形，但是已经无需进行切分
  assert(Grid.valence(Grid.face(he)) > 4);
  return Grid.insert_edge(Grid.prev_halfedge(he), Grid.next_halfedge(he));
}

point_projection()

输入三个点P,A,B，计算P在直线AB上的投影Q，返回 AQ/AB 的值。用于投影推进时判断矩形的形状。

// 计算p0在p1p2所在的直线上投影，与向量p1p2相比的倍数
double StripDivide::point_projection(HTPoint p0, HTPoint p1, HTPoint p2)
{
  HTPoint v0 = p0 - p1, v1 = p2 - p1;
  return (v0[0] * v1[0] + v0[1] * v1[1]) / (v1[0] * v1[0] + v1[1] * v1[1]);
}

connect()（单侧推进）

在切好的一层中连接上下侧的点。单侧推进只需要一个参数，即条状区域一侧的半边，从该半边出发依次连接出四边形。单侧推进会使三角形都出现在另一侧的边缘，在实际使用中效果不好，已废弃。

//! 传入条状区域一端的内侧半边,对该区域进行点连接(单侧推进)
void StripDivide::connect(HTHalfedge index)
{
  //后面是四边形或三角形时停止切分
  while (Grid.valence(Grid.face(index)) > 4)
  {
    index = connect_rec(index);
  }
}

connect()（双侧推进）

在切好的一层中连接上下侧的点。双侧推进需要传入条状区域两侧的半边，从两侧交替连接矩形，使多出的三角形留在中间部位。不能适用于上下两层点数相差大于1的情况。在点数相差大于1时，应使用投影推进。

//! 传入条状区域两端的内侧半边,对该区域进行点连接(双侧推进)
void StripDivide::connect(HTHalfedge index_up, HTHalfedge index_down)
{
  while (Grid.valence(Grid.face(index_up)) > 4)
  {
    index_up = connect_rec(index_up);
    if (Grid.valence(Grid.face(index_down)) > 4)
    {
      index_down = connect_rec(index_down);
    }
  }
}

connect()（投影推进）

在切好的一层中连接上下侧的点。投影推进只需传入条状区域一侧的半边，使用 point_rejection 计算半边起点在下侧的投影值。

//! 传入条状区域两端的内侧半边,对该区域进行点连接（投影判断）
void StripDivide::connect(HTHalfedge index)
{
  while (Grid.valence(Grid.face(index)) > 4)
  {
    //计算投影值
    double value = point_projection(Grid.position(Grid.from_vertex(index)),
      Grid.position(Grid.to_vertex(index)),
      Grid.position(Grid.to_vertex(Grid.next_halfedge(index))));
    
    if (value > 0.5)  //大于0.5，向下连接三角形
    {
      index = connect_tri(index);
    }
    else if (value < -0.5)  //小于-0.5，向上连接三角形
    {
      index = connect_tri(Grid.prev_halfedge(index));
    }
    else  //连接矩形
    {
      index = connect_rec(index);
    }
  }
}

结果展示与分析

基础剖分

基础剖分中的点连接过程包括了三种实现：

  • 单侧推进
    从一侧开始逐个连接四边形，所有三角形会出现在另一侧的边界，网格变形大，效果不好。
  • 双侧推进
    从两侧开始交替连接四边形，三角形会出现在网格的中间。效果较好，误差分配比较均匀，缺点是不适用于相邻两行相差多个点的情况。
  • 投影推进
    根据投影计算误差的大小来判断是否添加三角形，三角形均匀分布。会使网格趋于矩形，主要用于单向加密时处理两侧点数相差较大的情况。



分区剖分

为了控制侧边边缘的波浪纹，采取分区剖分的方法，使边缘区域为结构网格，中间部分采用基础剖分。

分层分区剖分

由于分区剖分是从顶边截取固定长度范围作为结构网格区域，越向底边靠近，结构网格的范围越小。为了控制整体网格的质量，缩小波浪纹的范围，可以采用先分层，每层分别做分区剖分的方式，使大部分网格都保持均匀的角度。

局部单向加密

在工件的应力集中处，往往要进行单向的加密以更好的捕捉某一个方向的特征。可以通过指定加密区的位置比例和加密比例，对网格的某一部分进行加密。

技术及实习总结

关于团队协作的思考

  在大型项目的团队协作中，合理的分工和高效的协作是非常重要的。为了进行合理的分工，需要在项目初期就对项目架构进行完整的分析，确定各模块的功能以及接口。为了进行团队协作，需要制定工作进度表，在各成员完成对应的工作后可以及时进行对接，进行查漏补缺。

  在本次项目中，我们就陷入了团队协作的误区。在初期，我们的观点是“都看一看，都做一做”，导致每个人的思路和架构都不相同，各自为战，在对接和沟通上浪费了大量的时间。这在今后的项目开发中是绝不应该出现的。

关于设计模式的思考

  在软件开发中，需求变动和调整架构是经常会发生的。因此，在设计代码时不应该把接口和功能写死，要提升接口的泛用性，降低代码之间的耦合度，为将来的升级留出余量。

  在本次的开发中，由于初期的设计不到位，接口泛用性差，面对需求提升时代码不能复用，导致了代码的多次重构，不仅使编写代码的时间大量延长，也为收尾阶段的流程整合和代码调试带来了许多困难。“纸上得来终觉浅”，要想将软件工程的知识真正应用到实践中，还需要大量的经验积累。
